# Quick Reference - Production Deployment

## 🚀 3-Step Deployment

### Step 1: Generate Credentials
```bash
python setup_credentials.py
```
→ Creates `SECRET_KEY` and `ADMIN_PASSWORD_HASH`

### Step 2: Set Environment Variables

**Option A: Using .env file (Local/Development)**
```bash
cat .env  # Verify file was created
```

**Option B: Using Render Dashboard (Production)**
1. Go to your service in Render
2. Settings → Environment
3. Add these variables:
   - `SECRET_KEY=<value from .env>`
   - `ADMIN_PASSWORD_HASH=<value from .env>`
   - `FLASK_HOST=0.0.0.0`
   - `FLASK_DEBUG=False`
   - `ADMIN_USERNAME=admin`
   - `CORS_ORIGINS=https://your-app.onrender.com`

### Step 3: Deploy
```bash
git add .
git commit -m "Production hardening"
git push origin main
# Render auto-deploys
```

## 🔑 Environment Variables Explained

```env
# Web Server
FLASK_HOST=0.0.0.0           # Allow external connections
FLASK_PORT=5000              # Port (Render sets its own)
FLASK_DEBUG=False            # NEVER True in production

# Authentication
ADMIN_USERNAME=admin          # Username for login
ADMIN_PASSWORD_HASH=<hash>   # Generated by setup_credentials.py

# Security
SECRET_KEY=<hex-string>      # Generated by setup_credentials.py
CORS_ORIGINS=*              # Allowed domains (restrict in production)

# System
NETWORK_INTERFACE=eth0       # For packet capture (unavailable on Render)
LOG_LEVEL=INFO              # Logging level
```

## 🔐 Security Highlights

✅ **What Changed:**
- Passwords hashed with PBKDF2-SHA256 (not plaintext)
- Secret keys from environment (not hardcoded)
- Debug mode disabled (not True)
- Localhost binding removed (now 0.0.0.0)
- CORS restricted (not open to all)
- HTTP security headers added (6 new headers)
- Input validation on login (prevent injection)
- Better error messages (no info disclosure)

## 🐛 Troubleshooting

### Login Not Working
```bash
# Verify password hash exists
echo $ADMIN_PASSWORD_HASH

# Generate new credentials
python setup_credentials.py

# Set in environment
export ADMIN_PASSWORD_HASH=<new-hash>
```

### Dashboard Shows Errors
```bash
# Check Render logs
# Settings → Logs (tail -100)

# Common issues:
# - ADMIN_PASSWORD_HASH not set
# - SECRET_KEY not set
# - Database connection failed
```

### Port Already in Use
```bash
# Use different port
FLASK_PORT=8000 python dashboard/app.py
```
